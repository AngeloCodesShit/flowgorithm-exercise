5 esercizi con consigli e soluzione commentata

01
Calcola il doppio
Semplice

Disegna il diagramma di flusso di un programma che legge un numero intero dall'utente e stampa a schermo il suo doppio.

  • Inizia sempre con il blocco Inizio (ovale) e concludi con Fine.
  • La lettura dell'input va in un parallelogramma, non in un rettangolo.
  • Il calcolo doppio = n * 2 è un'operazione di processo: usa il rettangolo.
  • La stampa del risultato è un output: anche qui parallelogramma.
  • In questo esercizio non servono cicli né decisioni: il flusso è completamente lineare.

Soluzione

Diagramma di flusso

flowchart TD A([Inizio]) A --> B[/Leggi n/] B --> C["doppio = n * 2"] C --> D[/Scrivi doppio/] D --> E([Fine])

Pseudocodice

INIZIO
  Dichiara n, doppio : Intero

  Leggi n
  doppio = n * 2
  Scrivi doppio
FINE
02
Pari o Dispari
Medio

Disegna il diagramma di flusso di un programma che chiede all'utente 5 numeri interi uno alla volta, usando un ciclo FOR. Per ognuno stampa se il numero è pari o dispari.

  • Il ciclo FOR si disegna con un esagono. Scrivi dentro: i = 0 a 4 per 5 iterazioni.
  • Dall'esagono escono due frecce: una etichettata Ripeti (corpo del ciclo) e una Termina (uscita).
  • Per verificare pari/dispari usa l'operatore mod: n mod 2 = 0 significa "n è pari".
  • Il rombo della decisione deve avere esattamente due uscite etichettate Vero e Falso.
  • Dopo i due rami usa un cerchio connettore per riunire il flusso prima di tornare all'esagono.

Soluzione

Diagramma di flusso

flowchart TD A([Inizio]) A --> B@{ shape: hex, label: "FOR i=0 a 4" } B -->|Ripeti| C[/Leggi n/] C --> D{"n mod 2 = 0 ?"} D -->|Vero| E[/Scrivi Pari/] D -->|Falso| F[/Scrivi Dispari/] E --> G((M)) F --> G G --> B B -->|Termina| H([Fine])

Pseudocodice

INIZIO
  Dichiara i, n : Intero

  PER i = 0 A 4:
    Leggi n
    SE n mod 2 = 0 ALLORA
      Scrivi "Pari"
    ALTRIMENTI
      Scrivi "Dispari"
    FINE SE
  FINE PER
FINE
03
Valutazione dei voti
Medio

Disegna il diagramma di flusso di un programma che chiede all'utente di inserire un voto (da 0 a 10) per 6 volte consecutive. Per ogni voto stampa "Promosso" se il voto è ≥ 6, "Rimandato" se è compreso tra 4 e 5 inclusi, "Bocciato" se è minore di 4.

  • Usa una struttura IF / ELSE IF / ELSE: controlla prima voto >= 6, poi voto >= 4. Nel ramo Falso del primo rombo sai già che il voto è sotto 6.
  • Non usare tre IF separati: con ELSE IF ci si ferma al primo ramo vero.
  • Tutti e tre i rami devono convergere in un cerchio connettore prima di tornare al ciclo.
  • Il ciclo va da i = 0 a 5 per ottenere esattamente 6 iterazioni.
  • Attenzione all'ordine dei controlli: se controllassi prima voto >= 4, un voto di 8 finirebbe nel ramo "Rimandato".

Soluzione

Diagramma di flusso

flowchart TD A([Inizio]) A --> B@{ shape: hex, label: "FOR i=0 a 5" } B -->|Ripeti| C[/Leggi voto/] C --> D{"voto >= 6 ?"} D -->|Vero| E[/Scrivi Promosso/] D -->|Falso| F{"voto >= 4 ?"} F -->|Vero| G[/Scrivi Rimandato/] F -->|Falso| H[/Scrivi Bocciato/] E --> I((M)) G --> I H --> I I --> B B -->|Termina| J([Fine])

Pseudocodice

INIZIO
  Dichiara i, voto : Intero

  PER i = 0 A 5:
    Leggi voto
    SE voto >= 6 ALLORA
      Scrivi "Promosso"
    ALTRIMENTI SE voto >= 4 ALLORA
      Scrivi "Rimandato"
    ALTRIMENTI
      Scrivi "Bocciato"
    FINE SE
  FINE PER
FINE
04
Calcola la somma
Medio

Disegna il diagramma di flusso di un programma che chiede all'utente 8 numeri interi tramite un ciclo FOR. Il programma tiene traccia della somma di tutti i numeri inseriti e, al termine del ciclo, la stampa a schermo.

  • L'accumulatore (somma) va inizializzato a 0 prima del ciclo: dentro verrebbe azzerato ad ogni iterazione.
  • L'aggiornamento somma = somma + n è un processo: usa il rettangolo dentro il corpo del ciclo.
  • La stampa della somma va fuori dal ciclo, dopo il blocco Termina dell'esagono.
  • In questo esercizio non serve nessun rombo: non ci sono condizioni da valutare.
  • Ciclo da i = 0 a 7 per ottenere esattamente 8 iterazioni.

Soluzione

Diagramma di flusso

flowchart TD A([Inizio]) A --> B["somma = 0"] B --> C@{ shape: hex, label: "FOR i=0 a 7" } C -->|Ripeti| D[/Leggi n/] D --> E["somma = somma + n"] E --> C C -->|Termina| F[/Scrivi somma/] F --> G([Fine])

Pseudocodice

INIZIO
  Dichiara i, n, somma : Intero

  somma = 0

  PER i = 0 A 7:
    Leggi n
    somma = somma + n
  FINE PER

  Scrivi somma
FINE
05
Semaforo a turni
Logica

Disegna il diagramma di flusso di un programma che simula un semaforo a tempo. Il ciclo FOR esegue 3 iterazioni, ognuna delle quali rappresenta un cambio di stato. L'utente inserisce il colore attuale (Verde, Giallo, Rosso) e il programma stampa quale sarà il colore successivo. Se l'input non corrisponde a nessuno dei tre colori validi, il programma stampa "Colore non riconosciuto" senza aggiornare nulla.

  • Ci sono 4 casi possibili (Verde, Giallo, Rosso, non valido): usa una catena di tre rombi annidati. L'ultimo ramo Falso copre il caso non riconosciuto.
  • Segui l'ordine logico del semaforo: Verde → Giallo → Rosso.
  • Tutti e quattro i rami di output devono convergere in un cerchio connettore prima di tornare all'esagono FOR.
  • La variabile colore è di tipo Stringa, non Intero: dichiarala correttamente.
  • Pensa a cosa succederebbe se l'utente scrivesse "verde" con la v minuscola: il confronto fallirebbe. In un programma reale andresti a normalizzare l'input prima del confronto.

Soluzione

Diagramma di flusso

flowchart TD A([Inizio]) A --> B@{ shape: hex, label: "FOR i=0 a 2" } B -->|Ripeti| C[/Leggi colore/] C --> D{"colore = Verde ?"} D -->|Vero| E[/Scrivi Giallo/] D -->|Falso| F{"colore = Giallo ?"} F -->|Vero| G[/Scrivi Rosso/] F -->|Falso| H{"colore = Rosso ?"} H -->|Vero| I[/Scrivi Verde/] H -->|Falso| J[/Scrivi non riconosciuto/] E --> K((M)) G --> K I --> K J --> K K --> B B -->|Termina| L([Fine])

Pseudocodice

INIZIO
  Dichiara i      : Intero
  Dichiara colore : Stringa

  PER i = 0 A 2:
    Leggi colore
    SE colore = "Verde" ALLORA
      Scrivi "Giallo"
    ALTRIMENTI SE colore = "Giallo" ALLORA
      Scrivi "Rosso"
    ALTRIMENTI SE colore = "Rosso" ALLORA
      Scrivi "Verde"
    ALTRIMENTI
      Scrivi "Colore non riconosciuto"
    FINE SE
  FINE PER
FINE
